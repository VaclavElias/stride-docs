## Meshes and Models

- [StrideProceduralModel](https://github.com/tebjan/StrideProceduralModel) - Example creating a fully procedural mesh and model at runtime in Stride
- [Get Vertices and Indices from a Mesh](https://gist.github.com/Doprez/3974122afee1cbddae289e16057026f3) - Gist showing how to extract vertex and index data from an existing Stride mesh
- [Procedural Tetrahedron Mesh](https://github.com/profan/XenkoByteSized/blob/master/XenkoByteSized/ProceduralMesh/TetrahedronMesh.cs) - XenkoByteSized snippet building a procedural tetrahedron mesh from scratch
- [StrideLOD](https://github.com/Xisrith/StrideLOD) - Level-of-detail system that swaps mesh complexity based on camera distance
- [Basic Stride LOD System](https://gist.github.com/Doprez/5327fbd7d8c371938088db81d82d153b) - Lightweight gist implementing a simple distance-based LOD swap in Stride
- [Stride.IK](https://github.com/flipdp/Stride.IK/tree/master) - Inverse kinematics implementation for Stride skeleton rigs
- [StrideTransformationInstancing](https://github.com/tebjan/StrideTransformationInstancing) - Demonstrates GPU instancing for rendering many transformed copies of a mesh efficiently
- [SuperCharacterController](https://github.com/corthax/SuperCharacterController) - Feature-rich character controller handling slopes, steps, and collision for Stride
- [Multi-Mesh Shader / Combined Meshes](https://github.com/profan/XenkoByteSized/blob/master/XenkoByteSized/ProceduralMesh/CombinedMeshes.cs) - XenkoByteSized snippet combining multiple meshes into a single draw call
